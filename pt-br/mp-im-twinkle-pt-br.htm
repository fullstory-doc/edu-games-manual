Twinkle

Telas

Descrição

O Twinkle é um cliente Voice over Internet Protocol (VoIP) para fazer chamadas usando SIP em uma rede IP. Dá suporte a ligações ponto a ponto ou a uma rede usando um proxy SIP para rastreamento de suas chamadas.

Além de ligações básicas, o Twinkle possui os seguintes recursos, independentemente do que o seu provedor de VoIP disponibiliza para você:

  • identificador de múltiplas chamadas
  • ringtones personalizados
  • chamada em espera
  • chamada em pausa
  • conferências em 3 vias
  • mudo
  • redirecionamento de chamadas
  • redirecionamento incondicional
  • redirecionamento quando ocupado
  • redirecionamento sem resposta
  • rejeição de redirecionamento
  • transferência mediante consulta (novo)
  • rejeição de transferência
  • rejeição de chamada
  • repetir última chamada
  • não perturbar
  • autorresposta
  • indicador de espera
  • discador de viva voz
  • scripts definidos pelo usuário para eventos de chamadas
  • por exemplo, para implementar rejeições seletivas ou alterar o som da campainha
  • eventos RFC 2833 DTMF
  • in-band DTMF
  • out-of-band DTMF (SIP INFO)
  • suporte STUN para NAT
  • mandar o NAT manter os pacotes acionados quando usando STUN
  • NAT através de provisionamento estático
  • indicador de chamadas perdidas
  • histórico de chamadas, com detalhes para chamadas recebidas, feitas, bem sucedidas e perdidas
  • suporte DNS SRV
  • redirecionamento automático a servidor alternativo se o usado estiver indisponível
  • outros programas podem originar uma chamada via Twinkle, por exemplo, chamadas de livros de endereço
  • ícone na área de notificação
  • menu na área de notificação, possibilitando chamar e atender enquanto o Twinkle permanece escondido
  • conversão de números definida pelo usuário
  • livro de endereços simples
  • suporte a UDP e TCP (novo) como transporte para o SIP
  • presença
  • mensagens instantâneas
  • transferidor de arquivo com mensageiro instantâneo simples
  • indicador de composição de mensagens
  • linha de comando (CLI)
    • Segurança VoIP
    • comunicação segura, pela ZRTP/SRTP
    • suporte a autenticação MD5 para qualquer chamada SIP
    • suporte a autenticação AKAv1-MD5 para qualquer chamada SIP (novo)
    • ocultação de identidade
    Codecs de áudio
    • G.711 A-law (64 kbps payload, 8 kHz sampling rate)
    • G.711 u-law (64 kbps payload, 8 kHz sampling rate)
    • GSM (13 kbps payload, 8 kHz sampling rate)
    • Speex de banda estreita (15.2 kbps payload, 8 kHz sampling rate)
    • Speex banda larga (28 kbps payload, 16 kHz sampling rate)
    • Speex ultra banda larga (36 kbps payload, 32 kHz sampling rate)
    • G.726 (16, 24, 32 ou 40 kbps payload, 8 kHz sampling rate)
    Para todos os codecs, estão disponíveis as seguintes opções de pré-processamento, para aumentar a qualidade de uma chamada:
    • controle automático de ganho (novo)
    • redução de ruído (novo)
    • detecção de atividade de voz (novo)
    • controle do eco [experimental] (novo)
